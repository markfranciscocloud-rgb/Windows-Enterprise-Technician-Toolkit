# Safety and Operational Rules

- Obtain authorization before collecting data from any device.
- Avoid elevation unless a specific diagnostic requires it.
- Do not assume a suspicious connection or process is malicious without corroborating evidence.
- Do not apply Microsoft or industry security baselines blindly; test them against the correct Windows release and organizational requirements.
- Preserve original logs and reports. Work from copies during analysis.
- Record collection time, user, hostname, purpose, and scope.
- Treat exported reports as potentially sensitive because they may contain usernames, IP addresses, software details, and device identifiers.
- Never commit generated `Logs/` or `Reports/` content to a public repository.
